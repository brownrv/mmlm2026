# GitHub Issues and Project Setup

This repository uses GitHub Issues as durable work memory for ideas, retests, and follow-ups.

## Issue Labels

Create these labels:

- `experiment`
- `idea`
- `retest`
- `data`
- `feature-engineering`
- `modeling`
- `simulation`
- `submission-strategy`
- `infra`
- `docs`
- `bug`

## Project Board Columns

Use this default column flow:

1. Inbox
2. Ready
3. Running
4. Review
5. Blocked
6. Revisit Later
7. Done

## Usage Rules

- New promising but incomplete ideas become Issues.
- If assumptions change, open/refresh retest Issues linked to impacted experiments/decisions.
- Every substantial PR should link at least one Issue.
